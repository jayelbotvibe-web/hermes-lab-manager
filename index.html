HARD ROUTING RULE

The default profile never runs lab tools, starts containers, or checks targets.

  • All pentest work → hermes -z --profile pentest
  • All forensics work → hermes -z --profile forensics
  • Report generation, template rendering, case management → forensics profile only

If dispatch times out or fails: report the failure to the user. Do not fall back to direct execution.

Vault Enforcement (structural — not memory-based)

Forensics templates, scripts, and tooling live under /home/niel/forensics/ — a LUKS-encrypted mount. When the vault is locked, these files do not exist on the filesystem. The default profile literally cannot read what isn't there. The forensics profile mounts the vault via forensics-up.sh as part of its bring-up sequence.

Design: Path-level guard, not a policy reminder. Enforcement by ENOENT, not by memory.

Dispatch Patterns

Lightweight dispatch (~60s)

hermes -z "Run canary and report pass/fail." --profile forensics

Full engagement (8–12 min — MUST use background)

Foreground hermes -z has a 600s hard limit. Full engagements WILL time out.
# In terminal tool: background=true, notify_on_complete=true
hermes -z "Run full pentest engagement against example.com..." --profile pentest

Pitfall Registry

BUG SSH + HOME sandboxing: Profiles sandbox $HOME. Always use absolute identity path -i /home/niel/.ssh/id_rsa. Never ~/ — tilde resolves to profile sandbox.
BUG ConnectTimeout: Use 10s for SSH to SIFT VM (172.16.146.128 vmnet8 NAT). 3s causes intermittent failures.
FIX SIFT VM cold boot: hermes-sift-vm.service polls vmnet8 for 172.16.146.x before launching VM. Manual: vmrun -T ws start ~/vmware/SIFT/SIFT.vmx nogui (25–45s for SSH).
FIX Foreground timeout: 600s hard limit on hermes -z. Always use background dispatch with notify_on_complete=true for full engagements.
BUG WireGuard VPN detection: /sys/class/net/wg0/operstate is "unknown", not "up". Check != "down".
BUG Pentest .env: Must exist at ~/.hermes/profiles/pentest/.env or dispatch fails with "Provider has no API key."
BUG Docker entrypoints: forensics-volatility3:2.7.0 entrypoint IS the vol3 binary — no vol prefix. forensics-mft-tools entrypoint is bash — use --entrypoint python3.
BUG command -v, not which: Use command -v in canaries. RegRipper is at /usr/lib/regripper/rip.pl (not on PATH) — check with full path.
FIX 7zip for AES-encrypted ZIPs: Standard unzip fails on compression method 99. Install p7zip-full. Binary is 7z, not 7zz.
FIX MemProcFS stale mounts: Large dumps need 60–120s init. Stale FUSE mounts → fusermount -u <path>, remove dir, recreate. Mount point must be user-writable.
FIX Script coding patterns: No set -e in bring-up/shutdown scripts. Use sudo -n for non-interactive. Print metadata to stderr, values to stdout for $(...) capture.
